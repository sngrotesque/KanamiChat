# Calabiyau (Kanami) — 项目规划文档（Project Plan）

本文件用于记录 Calabiyau（香奈美聊天机器人）的整体规划、大纲、设计方向与未来路线图。  
目标是让项目结构清晰、可扩展，并便于长期维护。

# 1. 项目愿景（Vision）

Calabiyau 是一个跨平台的虚拟角色聊天应用，核心目标是：

- 让用户与“香奈美”进行自然、温柔、陪伴式的互动  
- 提供沉浸式 UI（背景、头像、称呼可自定义）  
- 支持基础聊天、语音、拍照互动等功能  
- 未来可扩展为“虚拟伴侣 + AI Agent”式应用

# 2. 产品定位（Positioning）

- **类型**：虚拟角色聊天应用  
- **角色**：香奈美（Kanami）  
- **风格**：温柔、可爱、陪伴型  
- **目标用户**：喜欢二次元角色、虚拟陪伴、轻量聊天体验的用户  
- **平台**：Windows / Linux / Android（未来可扩展）

# 3. 当前功能结构（Feature Overview）

## 3.1 页面布局

### ① 主页（Home）
- 背景图为香奈美  
- 展示公告、提示、活动等信息  
- 未来可加入动态背景、角色动画  

### ② 聊天（Chat）
- 整体 UI 参考 QQ / 微信  
- 背景图可自定义  
- 支持文字聊天  
- 支持语音消息  
- 未来可加入表情包、贴纸、富文本  

### ③ 我的（Profile）
- 更换香奈美头像  
- 更换用户头像  
- 设置香奈美对用户的称呼  
- 未来可加入主题、配色、角色语音包等

# 4. 功能规划（Feature Plan）

## 4.1 已实现 / 正在开发
- 基础文字聊天  
- 语音消息  
- 拍照发送给香奈美  
- 多平台支持（Win / Linux / Android）

## 4.2 计划中的功能
- 语音通话（视技术难度与稳定性决定）  
- 自定义角色语音（TTS）  
- 香奈美的“主动对话”模式  
- 本地缓存与聊天记录管理  
- 多角色系统（未来可扩展）  

## 4.3 未来可能加入
- 动态 / 3D 角色  
- 插件系统（让用户扩展香奈美能力）  
- 云端同步  
- AI 记忆系统  
- 场景模式（如：睡前聊天、学习陪伴）

# 5. 技术架构（Technical Overview）

## 5.1 技术栈
- Flutter（跨平台 UI）  
- Dart（业务逻辑）  
- 平台相关插件（相机、麦克风、文件系统）  

## 5.2 模块划分
- UI 层  
- 业务逻辑层  
- AI 对话模块  
- 多媒体模块（语音、相机）  
- 配置与存储模块
